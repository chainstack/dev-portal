The `transactionsByMessage` endpoint retrieves a list of transactions related to a specific message in the TON blockchain. This endpoint allows you to fetch transactions associated with a particular message hash and provides options for pagination.

## Parameters

* `direction` (string, required) тАФ The direction of the message. Possible values: `in` or `out`. Default: `out`.
* `msg_hash` (string, required) тАФ The hash of the message. Example: `po4/F9OgHrJGTUCtobScZrW/gOwVPj7s3f40HV32EII=`.
* `limit` (integer, optional) тАФ The maximum number of transactions to return. Default: `128`.
* `offset` (integer, optional) тАФ The number of transactions to skip before starting to return results. Default: `0`.

## Response

* `transactions` (array) тАФ An array of transaction objects, each containing:
  + `hash` (string) тАФ The transaction hash.
  + `lt` (string) тАФ The logical time of the transaction.
  + `account` (string) тАФ The account address involved in the transaction.
  + `now` (integer) тАФ The timestamp of the transaction.
  + `origStatus` (string) тАФ The original status of the account.
  + `endStatus` (string) тАФ The end status of the account.
  + `totalFees` (string) тАФ The total fees for the transaction.

## Use case

The `transactionsByMessage` endpoint is useful for various applications that need to track or analyze specific messages and their related transactions:

1. Wallet applications can use this to track the status of sent or received messages and their associated transactions.
2. Block explorers can provide detailed information about message propagation and the resulting transactions.
3. Developers can use this endpoint to debug and verify message processing in smart contracts.
4. Analytics tools can track the impact of specific messages on the blockchain state.
