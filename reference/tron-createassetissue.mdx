TRON API method that creates an unsigned transaction for issuing a new TRC10 token on the TRON network. TRC10 tokens are native TRON assets that can be created without deploying smart contracts.

## Parameters

* `owner_address` ‚Äî the address that will issue and own the token
* `name` ‚Äî the full name of the token (string)
* `abbr` ‚Äî the token symbol or abbreviation (string)
* `total_supply` ‚Äî the total number of tokens to be issued (integer)
* `trx_num` ‚Äî the TRX amount used in the exchange rate calculation (integer)
* `num` ‚Äî the token amount used in the exchange rate calculation (integer)
* `precision` ‚Äî the number of decimal places supported by the token (integer, 0-6)
* `start_time` ‚Äî ICO start timestamp in milliseconds (integer)
* `end_time` ‚Äî ICO end timestamp in milliseconds (integer)
* `description` ‚Äî detailed description of the token project (string)
* `url` ‚Äî official website URL for the token project (string)
* `free_asset_net_limit` ‚Äî bandwidth provided by token issuer for token operations (integer, optional)
* `public_free_asset_net_limit` ‚Äî public bandwidth limit available for this token (integer, optional)
* `frozen_supply` ‚Äî array of frozen supply configurations (optional)
* `visible` ‚Äî optional boolean parameter. When set to `true`, addresses are in base58 format. Default is `false`.

## Response

* `visible` ‚Äî indicates the address format used in the response
* `txID` ‚Äî the transaction hash
* `raw_data` ‚Äî raw transaction data including:
  * `contract` ‚Äî array containing the asset issue contract
  * `ref_block_bytes` ‚Äî reference block bytes
  * `ref_block_hash` ‚Äî reference block hash  
  * `expiration` ‚Äî transaction expiration timestamp
  * `timestamp` ‚Äî transaction creation timestamp
  * `fee_limit` ‚Äî maximum fee allowed for this transaction
* `raw_data_hex` ‚Äî hexadecimal representation of the raw transaction

## Use case

The `wallet/createassetissue` method is used for:

* Creating new TRC10 tokens for projects and organizations
* Launching Initial Coin Offerings (ICOs) and token sales
* Building token issuance functionality in wallet applications
* Developing automated token creation systems for DeFi platforms
