Robinhood Chain API method that traces the execution of all transactions in a block identified by its number. This method provides detailed traces for every transaction and call in the block, including gas usage and internal call information.

<Note>
Learn how to [deploy a node](/docs/debug-and-trace-apis#robinhood) with the debug and trace API methods enabled.
</Note>

<Warning>
This method is available for post-Nitro blocks only (block 22,207,815 and later). For pre-Nitro blocks, use `arbtrace_block` instead.
</Warning>

## Parameters

* `quantity or tag` ‚Äî the block number in hex format or block tag:
  + `latest` ‚Äî the most recent block.
  + `safe` ‚Äî the block that received justification from the beacon chain.
  + `finalized` ‚Äî the block accepted as canonical by more than 2/3 of validators.
  + `earliest` ‚Äî the earliest available or genesis block.
  + `pending` ‚Äî the pending state and transactions block.

* `tracer` ‚Äî an object identifying the type of tracer and its configuration:
  + `4byteTracer` ‚Äî tracer that captures the function signatures and call data sizes for all functions executed during a transaction.
  + `callTracer` ‚Äî tracer that captures information on all call frames executed during a transaction.
  + `prestateTracer` ‚Äî tracer with two modes: `prestate` and `diff`, returning account states needed to execute transactions.

## Response types

### `4byteTracer` response

* `object` ‚Äî the `4byteTracer` traces object:
  + `result` ‚Äî a map of the function signature, the call data size, and how many times the function was called.

### `callTracer` response

* `object` ‚Äî the `callTracer` traces object:
  + `from` ‚Äî the address of the sender who initiated the transaction.
  + `gas` ‚Äî the units of gas included in the transaction by the sender.
  + `gasUsed` ‚Äî the total used gas by the call, encoded as hexadecimal.
  + `to` ‚Äî the address of the recipient of the transaction if it was a transaction to an address. For contract creation transactions, this field is `null`.
  + `input` ‚Äî the optional input data sent with the transaction.
  + `output` ‚Äî the return value of the call, encoded as a hexadecimal string.
  + `error` ‚Äî an error message in case the execution failed.
  + `revertReason` ‚Äî the reason why the transaction was reverted, returned by the smart contract if any.
  + `calls` ‚Äî a list of sub-calls made by the contract during the call, each represented as a nested call frame object.

### `prestateTracer` response

* `object` ‚Äî the `prestateTracer` traces object:
  + `smart contract address` ‚Äî the address of the smart contract associated with the result.
    - `balance` ‚Äî the balance of the contract, expressed in Wei and encoded as a hexadecimal string.
    - `code` ‚Äî the bytecode of the contract, encoded as a hexadecimal string.
    - `nonce` ‚Äî the nonce of the account associated with the contract, represented as an unsigned integer.
    - `storage` ‚Äî a map of key-value pairs representing the storage slots of the contract.

## `debug_traceBlockByNumber` code examples

<CodeGroup>
```javascript ethers.js
const ethers = require('ethers');
const NODE_URL = "YOUR_CHAINSTACK_ENDPOINT";
const provider = new ethers.JsonRpcProvider(NODE_URL);

const traceBlockByNumber = async (block) => {
  const tracer = { tracer: "callTracer" };
  const traces = await provider.send("debug_traceBlockByNumber", [block, tracer]);
  console.log(traces);
};

traceBlockByNumber("latest");
```

```python web3.py
from web3 import Web3
node_url = "YOUR_CHAINSTACK_ENDPOINT"
web3 = Web3(Web3.HTTPProvider(node_url))

block = "latest"

tracer = {"tracer": "callTracer"}
block_traces = web3.provider.make_request('debug_traceBlockByNumber', [block, tracer])
print(block_traces)
```
</CodeGroup>


## Use case

The `debug_traceBlockByNumber` method is useful for analyzing all transactions in a block by number. Developers can use it to calculate average gas usage across transactions, monitor smart contract activity patterns, or build block-level analytics dashboards.
