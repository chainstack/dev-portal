The `nft/items` endpoint retrieves a list of NFT (Non-Fungible Token) items from the TON blockchain. This endpoint allows you to fetch information about individual NFT items and provides options for pagination.

## Parameters

* `address` (array of strings, optional) ‚Äî NFT item address in any form. Max: 1000.
* `owner_address` (array of strings, optional) ‚Äî Address of NFT item owner in any form. Max: 1000.
* `collection_address` (array of strings, optional) ‚Äî Collection address in any form.
* `index` (array of strings, optional) ‚Äî Index of item for given collection. Max: 1000.
* `sort_by_last_transaction_lt` (boolean, optional) ‚Äî Sort NFT items by last transaction lt descending. **Warning:** results may be inconsistent during pagination with limit and offset.
* `limit` (integer, optional) ‚Äî The maximum number of NFT items to return. Default: `10`.
* `offset` (integer, optional) ‚Äî The number of NFT items to skip before starting to return results. Default: `0`.

## Response

* `nft_items` (array) ‚Äî An array of NFT item objects, each containing:
  + `address` (string) ‚Äî The address of the NFT item.
  + `auction_contract_address` (string) ‚Äî The address of the auction contract, if applicable.
  + `code_hash` (string) ‚Äî The code hash of the NFT item.
  + `collection` (object) ‚Äî Information about the collection this item belongs to:
    - `address` (string) ‚Äî The address of the collection.
    - `code_hash` (string) ‚Äî The code hash of the collection.
    - `collection_content` (object) ‚Äî The content/metadata of the collection.
    - `data_hash` (string) ‚Äî The data hash of the collection.
    - `last_transaction_lt` (string) ‚Äî The logical time of the last transaction.
    - `next_item_index` (string) ‚Äî The index of the next item in the collection.
    - `owner_address` (string) ‚Äî The address of the collection owner.
  + `collection_address` (string) ‚Äî The address of the collection.
  + `content` (object) ‚Äî The content of the NFT item.
  + `data_hash` (string) ‚Äî The data hash of the NFT item.
  + `index` (string) ‚Äî The index of the NFT item within its collection.
  + `init` (boolean) ‚Äî Whether the NFT item is initialized.
  + `last_transaction_lt` (string) ‚Äî The logical time of the last transaction.
  + `on_sale` (boolean) ‚Äî Whether the NFT item is on sale.
  + `owner_address` (string) ‚Äî The address of the current owner.
  + `real_owner` (string) ‚Äî The address of the real owner (if different from owner_address, e.g. when on sale).
  + `sale_contract_address` (string) ‚Äî The address of the sale contract, if applicable.
* `address_book` (object) ‚Äî Address book information.
* `metadata` (object) ‚Äî Metadata information.

## Use case

The `nft/items` endpoint is useful for various applications that need to interact with or display information about individual NFT items on the TON blockchain:

1. NFT marketplaces can use this to list available NFTs for browsing or trading.
2. Wallet applications can display NFT items owned by a user or allow users to explore popular NFTs.
3. Analytics platforms can gather data on NFT items, such as ownership distribution, rarity, and metadata patterns.
4. Developers can use this endpoint to integrate NFT item information into their dApps or games.
5. Collectors can verify the on-chain status and details of their owned NFTs.
6. Artists and creators can check the current status and ownership of their minted NFTs.
