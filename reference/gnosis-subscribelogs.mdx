Gnosis Chain subscription equivalent to [eth\_newFilter](/reference/gnosis-newfilter). `subscribe("logs")` allows developers to subscribe to real-time updates about new event logs on the Gnosis Chain; the application will receive notifications whenever new logs matching the filter are emitted.

## Parameters

* `string` ‚Äî a keyword identifying the type of event to subscribe to, `logs` in this case.

* `object` ‚Äî (optional) The subscription options:

  * `fromBlock` ‚Äî an integer that specifies the starting block number from which the logs should be fetched. `null` by default.
  * `address` ‚Äî the contract address from which the logs should be fetched. It can be a single address or an array of addresses.
  * `topics` ‚Äî an array of `DATA` topics. The event topics for which the logs should be fetched. It can be a single topic or an array of topics.

* `function` ‚Äî (optional) a callback function that will be called every time a new event of the specified type is received. This function takes two parameters: `error` and `result`. The error parameter contains any error that occurred while subscribing to the event, and the result parameter contains the data for the event that was received.

## Response

* `array` ‚Äî an array of log objects that match the specified filter:

  * `address` ‚Äî the contract address from which the event originated.
  * `topics` ‚Äî an array of 32-byte data fields containing indexed event parameters.
  * `data` ‚Äî the non-indexed data that was emitted along with the event.
  * `blocknumber` ‚Äî the block number in which the event was included. `null` if it is pending.
  * `transactionhash` ‚Äî the hash of the transaction that triggered the event. `null` if pending.
  * `transactionindex` ‚Äî the integer index of the transaction within the block's list of transactions. `null` if it is pending.
  * `blockhash` ‚Äî the hash of the block in which the event was included. `null` if it is pending.
  * `logindex` ‚Äî the integer identifying the index of the event within the block's list of events. `null` if pending.
  * `removed` ‚Äî the boolean value indicating if the event was removed from the blockchain due to a chain reorganization. `True` if the log was removed. `False` if it is a valid log.

## `subscribe("logs")` code example

<Info>
  Note that subscriptions require a WebSocket connection.
</Info>

Use the ethers.js `WebSocketProvider` to attach event listeners to a log filter:

* `provider.on(filter, listener)` ‚Äî registers a listener that activates for each new log matching the filter.
* `provider.on("error", listener)` ‚Äî activates if an error is detected during the subscription.
* `provider.off(filter)` ‚Äî removes the listener and stops receiving updates for the filter.

<CodeGroup>
  ```javascript index.js
  const { ethers } = require("ethers");
  const NODE_URL = "CHAINSTACK_WSS_URL";
  const provider = new ethers.WebSocketProvider(NODE_URL);

  // Only log transfer events from the given smart contract.
  const logsFilter = {
      address: '0x0d500B1d8E8eF31E21C99d1Db9A6444d3ADf1270',
      topics: ['0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef']
  }

  async function subscribeToLogs() {
      try {
          // Create a new subscription to logs matching the filter
          provider.on(logsFilter, handleLogs);

          // Attach an event listener for subscription errors
          provider.on("error", handleError);

          console.log("Subscribed to logs");
      } catch (error) {
          console.error(`Error subscribing to logs: ${error}`);
      }
  }

  /* Fallback functions to react to the different events */

  // Event listener that logs the filtered events
  async function handleLogs(log) {
      console.log(log)
  }

  // Event listener that logs any errors that occur
  function handleError(error) {
      console.error(`Error receiving new logs: ${error}`);
  }

  subscribeToLogs();
  ```
</CodeGroup>

## Use case

A practical use case for `subscribe("logs")` is a DApp that continuously listens for new specific event logs and isolates certain fields for analytics purposes. This is useful, for example, to track events about a specific ERC-721 token.
