The `tryLocateSourceTx` method attempts to locate the source transaction in the TON blockchain based on the source address, destination address, and the logical time of a result transaction. This method is useful for tracing back the origin of a transaction.

## Parameters

* `source` (string, required) ‚Äî The source address of the result transaction. Example: `UQAX2d4C2d48JeA-0y2_peGabRVumnKY22xPvkuEayC3zTI1`.
* `destination` (string, required) ‚Äî The destination address of the result transaction. Example: `UQBP4mzpDIywL1SV-Wp9ZuBBlzprR9eXQgSYGEXiUEHm73vA`.
* `created_lt` (string, required) ‚Äî The logical time when the result transaction was created. Example: `47597573000002`.

## Response

* `block` (object) ‚Äî Information about the block containing the source transaction:
  + `workchain` (integer) ‚Äî The workchain ID of the block.
  + `shard` (string) ‚Äî The shard ID of the block.
  + `seqno` (integer) ‚Äî The sequence number of the block.
  + `root_hash` (string) ‚Äî The root hash of the block.
  + `file_hash` (string) ‚Äî The file hash of the block.
* `transaction_id` (object) ‚Äî The identifier of the source transaction:
  + `lt` (string) ‚Äî The logical time of the source transaction.
  + `hash` (string) ‚Äî The hash of the source transaction.

## Use case

A possible use case for the `tryLocateSourceTx` method in TON is for applications or services that need to trace the origin of transactions. This method can be used to:

1. Implement transaction history tracing in blockchain explorers.
2. Analyze the chain of events leading to a specific transaction.
3. Verify the authenticity of a transaction by tracing it back to its source.
4. Debug smart contract interactions by identifying the initiating transaction.
