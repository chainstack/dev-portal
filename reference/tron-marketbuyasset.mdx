## Parameters

* `owner_address` ‚Äî address of the account placing the buy order (base58 with `visible: true`, or hex with `visible: false`).
* `sell_token_id` ‚Äî TRC10 token ID of the asset offered, as a numeric string.
* `sell_token_quantity` ‚Äî integer amount to offer (token base units; if TRX, sun).
* `buy_token_id` ‚Äî TRC10 token ID of the asset to purchase, as a numeric string.
* `buy_token_quantity` ‚Äî integer desired amount to purchase (token base units; if TRX, sun).

## Response

* `visible` ‚Äî boolean indicating whether addresses are in visible format
* `txID` ‚Äî unique transaction ID for the buy order transaction
* `raw_data` ‚Äî raw transaction data containing:
  * `contract` ‚Äî array with market buy order contract details
  * `ref_block_bytes` ‚Äî reference block bytes for transaction validation
  * `ref_block_hash` ‚Äî hash of the reference block
  * `expiration` ‚Äî transaction expiration timestamp
  * `timestamp` ‚Äî transaction creation timestamp
* `raw_data_hex` ‚Äî complete transaction data encoded in hexadecimal format

## Use case

The `wallet/marketbuyasset` method is used for:

* Creating buy orders to acquire tokens at desired prices.
* Providing demand liquidity to the TRON DEX marketplace.
* Implementing automated token acquisition strategies.
* Building custom trading bots and arbitrage systems.
* Enabling decentralized token exchange without centralized intermediaries (legacy orderbook).
