# Account States

The `accountStates` endpoint queries account states from the TON blockchain, including historical state data. This endpoint provides detailed information about account balances, status, and optionally the code and data cells.

## Parameters

* `account_address` (string, optional) ‚Äî Account address to query.
* `include_boc` (boolean, optional) ‚Äî Include code and data BoC in response. Default: `false`.
* `limit` (integer, optional) ‚Äî Maximum number of states to return. Default: `10`.
* `offset` (integer, optional) ‚Äî Number of states to skip for pagination. Default: `0`.
* `sort` (string, optional) ‚Äî Sort order: `asc` or `desc`. Default: `desc`.

## Response

* `account_states` (array) ‚Äî Array of account state objects:
  * `account` (string) ‚Äî Account address.
  * `hash` (string) ‚Äî State hash.
  * `balance` (string) ‚Äî Account balance in nanotons.
  * `account_status` (string) ‚Äî Account status (`active`, `uninit`, `frozen`).
  * `last_trans_lt` (integer) ‚Äî Logical time of last transaction.
  * `last_trans_hash` (string) ‚Äî Hash of last transaction.
  * `frozen_hash` (string) ‚Äî Hash if account is frozen.
  * `code_hash` (string) ‚Äî Hash of the code cell.
  * `data_hash` (string) ‚Äî Hash of the data cell.
  * `code_boc` (string) ‚Äî Code BoC (if `include_boc` is true).
  * `data_boc` (string) ‚Äî Data BoC (if `include_boc` is true).

* `address_book` (object) ‚Äî Address book mapping.

## Use case

The `accountStates` endpoint is useful for applications that need detailed account information:

1. Block explorers displaying account history and state changes.
2. Smart contract analysis tools examining code and data.
3. Wallet applications showing detailed account information.
4. Auditing tools tracking account state over time.
5. Research applications studying account behavior patterns.
