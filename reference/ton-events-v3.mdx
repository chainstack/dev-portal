# Events

The `events` endpoint queries blockchain events, which combine traces with their decoded actions. Events provide a complete view of what happened in a transaction flow, including all resulting actions.

## Parameters

* `account` (string, optional) ‚Äî Filter by account address.
* `start_utime` (integer, optional) ‚Äî Start of time range (unix timestamp).
* `end_utime` (integer, optional) ‚Äî End of time range (unix timestamp).
* `limit` (integer, optional) ‚Äî Maximum number of events to return. Default: `10`.
* `offset` (integer, optional) ‚Äî Number of events to skip for pagination. Default: `0`.
* `sort` (string, optional) ‚Äî Sort order: `asc` or `desc`. Default: `desc`.

## Response

* `events` (array) ‚Äî Array of event objects:
  * `trace_id` (string) ‚Äî Trace ID.
  * `external_hash` (string) ‚Äî Hash of the external message.
  * `mc_seqno_start` (string) ‚Äî Start masterchain sequence number.
  * `mc_seqno_end` (string) ‚Äî End masterchain sequence number.
  * `start_lt` (string) ‚Äî Start logical time.
  * `start_utime` (integer) ‚Äî Start unix timestamp.
  * `end_lt` (string) ‚Äî End logical time.
  * `end_utime` (integer) ‚Äî End unix timestamp.
  * `trace_info` (object) ‚Äî Trace state and statistics.
  * `actions` (array) ‚Äî Decoded actions from the event.
  * `is_incomplete` (boolean) ‚Äî Whether the event is still processing.

* `address_book` (object) ‚Äî Address book mapping.

## Use case

The `events` endpoint is useful for applications that need combined trace and action data:

1. Building activity feeds showing user transaction history.
2. Notification systems alerting on specific events.
3. Analytics dashboards tracking blockchain activity.
4. Block explorers showing transaction details with decoded actions.
5. Debugging complex multi-contract interactions.

Here's an example of getting recent events:

<CodeGroup>
  ```shell shell
  curl -X GET \
    'https://ton-mainnet.core.chainstack.com/f2a2411bce1e54a2658f2710cd7969c3/api/v3/events?limit=5' \
  ```

</CodeGroup>

<Tip>
  Events combine the structure of [traces](/reference/ton-traces-v3) with the decoded information from [actions](/reference/ton-actions-v3), making them ideal for building user-facing transaction histories.
</Tip>
